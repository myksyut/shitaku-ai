# プロジェクトコンテキスト

## プロジェクト概要

- **プロダクト名**: Shitaku.ai（支度.ai）
- **解決する問題**: 定例MTGのアジェンダ準備が煩雑。前回の議論内容を思い出せない、MTG外の進捗（Slack）を把握できない、社内固有名詞の表記揺れで文脈が失われる
- **ターゲットユーザー**: PM・EM（3〜10本の定例MTGを抱える）。MVP検証対象は5〜10人
- **利用シーン**: MTG前に「次回アジェンダ提案」ボタンを押し、前回議事録＋Slack履歴から自動生成されたアジェンダを確認・編集してMTGに臨む

## 開発体制

- **チーム構成**: 個人開発（self-funded）
- **開発フェーズ**: MVP開発着手前（2026-01-31時点）
- **外部リソース**: AWS Activate申請済み（$1,000クレジット待ち）、既存$100クレジット残

## ビジネス制約

1. **コストゼロ運用**: Vercel Hobby無料枠 + Supabase無料枠 + AWS Activateクレジットで月額$0を維持
2. **ユビキタス言語辞書必須**: 文字起こしの固有名詞ブレが致命的（NotebookLM検証で判明）。正規化なしでは実用不可
3. **確定的情報でMVP成立**: アジェンダ生成は「前回議事録＋Slack期間指定＋辞書」のみで成立。ベクトル検索（pgvector）はP1の補強機能

## コア機能（P0）

| # | 機能 | 概要 |
|---|------|------|
| F1 | MTGエージェント作成 | 定例MTGシリーズごとにエージェントを作成 |
| F2 | 議事録アップロード＋正規化 | 辞書で固有名詞を正規化してDB保存 |
| F3 | ユビキタス言語辞書（CRUD） | 人名・プロジェクト・用語・顧客・略語を管理 |
| F4 | Slack連携 | OAuth → チャンネル選択 → MTG間メッセージ取得 |
| F5 | アジェンダ自動生成 | 確定的情報収集 → Claude Haiku 4.5で生成 |

## ドメイン知識

### ユビキタス言語辞書のデータ構造
- **正式名称**（必須）: 金沢太郎
- **カテゴリ**（必須）: 人名 / プロジェクト / 用語 / 顧客 / 略語
- **表記揺れ**（必須・配列）: ["かなざわ", "金澤", "Kanazawa"]
- **説明・役割**（任意）: フロントエンド担当、チームA所属

### アジェンダ生成フロー
1. 前回議事録（agent_id + 最新created_at）を取得
2. 前回MTG〜今日のSlack履歴（日付範囲フィルタ）を取得
3. ユビキタス言語辞書（全件）を参照用に取得
4. Claude Haiku 4.5でアジェンダ生成
